这套导航系统是一个基于 PHP + HTML + JSON 构建的轻量级、响应式网站导航及发布页系统。它具有前后端分离的逻辑架构，支持通过后台管理界面动态修改内容，并具备自动测速（Ping）功能。

以下是对该系统的详细描述与技术讲解：

1. 系统整体架构
系统的运行逻辑主要围绕 config.json 配置文件展开。后端负责修改此文件，前端负责读取并渲染。

前端渲染层 (index.html)：用户访问的首页，负责展示数据、执行测速逻辑及适配移动端。

后端管理层 (index.php, auth.php)：管理员使用的控制台，负责身份验证及修改配置。

数据存储层 (config.json)：存储所有的站点名称、链接、样式、图片路径等配置信息。

API 接口层 (api.php)：为前端提供统一的 JSON 数据接口。

2. 核心文件讲解
① index.html (用户展示页)
这是系统的主体，采用原生的 JavaScript 和 CSS 变量（CSS Variables）实现。

动态渲染：通过 fetch 请求 api.php 获取配置，动态生成 Logo、标题、轮播图和线路列表。

自动测速 (Ping)：系统内置了一个 testPing 函数，通过 fetch 的 HEAD 请求测试各个线路的延迟，并根据延迟高低（绿/橙/红）动态更新 UI 状态。

响应式适配：支持 PC 端和手机端不同的背景图加载，并根据屏幕尺寸自动切换。

② index.php & auth.php (后台管理)
身份验证：auth.php 定义了管理员的用户名（默认 admin）和密码（默认 admin123）。

可视化配置：提供了直观的界面来管理“主标题组”、“副标题组”、“轮播图”和“线路入口”。

资源管理：支持上传 Logo、PC 背景、手机背景和客服二维码，上传的文件会自动存放在 /img/ 目录下。

③ config.json (核心数据)
这是整个系统的“大脑”。

它以 JSON 格式存储了所有的个性化设置。

示例结构：包含 titles（标题数组，含颜色和字号）、links（线路链接）、carousel（轮播图配置）等字段。

④ api.php (数据中转)
这是一个简单的 PHP 脚本，负责读取 admin/config.json 的内容并以 JSON 格式输出，同时处理了资源路径的修复逻辑，确保前端能正确加载图片。

3. 系统功能特点
高度自定义：无需修改代码，即可在后台调整边框宽度、圆角、背景透明度、文字描边等视觉参数。

测速功能：前端实时显示线路延迟，帮助用户选择响应最快的访问入口。

多媒体支持：背景支持图片或视频（MP4/WebM），提升视觉冲击力。

轻量无数据库：系统不使用 MySQL，迁移非常方便，只需拷贝整个文件夹到 PHP 环境即可运行。

4. 关键逻辑流程
加载阶段：用户打开 index.html -> JS 请求 api.php -> PHP 读取 config.json 返回数据。

渲染阶段：前端根据返回的 JSON 生成 HTML 结构，设置 CSS 变量。

测速阶段：JS 遍历 links 列表，对每个 URL 发起异步探测，计算耗时并显示在页面上。

更新阶段：管理员在 index.php 提交表单 -> PHP 将新数据写入 config.json -> 前台刷新即刻生效。
